# 沪深北 Deal Feedback Skill —— 投行视角的审核反馈分析

## 用途

抓取并解析三家交易所（北交所/上交所/深交所）三大业务类型（IPO/再融资/重大资产重组）的审核反馈材料，**从投行视角**输出结构化分析。

### 业务覆盖矩阵

| 业务类型 | 北交所 BSE | 上交所 SSE | 深交所 SZSE |
|----------|-----------|-----------|-----------|
| **IPO 首发** | 问询函 / 回复 / 招股说明书（注册稿） | 回复 / 招股说明书（注册稿） | 回复 / 招股说明书（注册稿） |
| **再融资**（定增/配股/可转债） | 问询函 / 回复 / 募集说明书 | 待实现（API 需 reverse engineering） | 待实现（API 需 reverse engineering） |
| **重大资产重组**（发行股份购买资产） | 问询函 / 回复 / 重组报告书 | 待实现 | 待实现 |

> **当前状态**: 北交所三类业务全部覆盖；上交所/深交所仅 IPO 已实现，再融资和重组的 API endpoint 待后续 reverse engineer 补充。上交所/深交所不公开问询函原件，只发布回复+主文件。

## CLI 调用

CLI 全局命令：`sszb-feedback`

### 常用命令

```bash
# 抓昨天的北交所 IPO（默认）
sszb-feedback fetch

# 抓近 7 天三家交易所 IPO
sszb-feedback fetch --exchange all --days 7

# 抓近 14 天北交所再融资项目
sszb-feedback fetch --exchange bse --business-type refinance --days 14

# 抓近 30 天北交所重大资产重组
sszb-feedback fetch --exchange bse --business-type asset_purchase --days 30

# 抓三家三业务全部，JSON 输出（喂给 LLM 写报告）
sszb-feedback fetch --exchange all --business-type all --days 7 --format json

# 只列文件清单，不下载 PDF（最快）
sszb-feedback fetch --exchange all --days 7 --no-download
```

### 参数

| 参数 | 取值 | 说明 |
|------|------|------|
| `--exchange / -e` | bse / sse / szse / all | 交易所，默认 bse |
| `--business-type / -b` | ipo / refinance / asset_purchase / all | 业务类型，默认 ipo |
| `--days / -d` | 1–40 | 回看天数，默认 1（昨天） |
| `--format / -f` | markdown / json | 输出格式 |
| `--no-download` | flag | 不下 PDF |
| `--no-parse` | flag | 下 PDF 但不抽文字 |

## 文档结构提示（写报告时定位关键信息用）

### 一、问询函 / 反馈意见 / 审核问询函

**北交所结构**：
1. **开头**：抬头致发行人 + 保荐机构，要求 20 个工作日内回复
2. **正文**：编号 "问题 1"、"问题 2" 等，每一问含：
   - 问题小标题（直接是风险信号关键词）
   - 背景信息（援引申报材料）
   - 具体要求（"请发行人：（1）…（2）…"）
3. **结尾**：要求保荐机构、申报会计师核查并发表意见

**抽取方法**：按 "问题 N" 切分。每一问的小标题就是核心信号，无需读完整问题。

### 二、问询回复 / 反馈回复

**结构**：
1. 开头：回复对照说明
2. 正文：按 "问题 1""问题 2" 编号，与问询函逐一对应
3. **关键标记**：`【回复】` 后紧跟公司核心回复内容

**抽取方法**：找 `【回复】` 标记，抽取核心结论与关键数据（金额、比例、笔数、时点）。

### 三、招股说明书（IPO 注册稿）—— 概览章节

| 内容 | 北交所 BSE | 上交所 SSE | 深交所 SZSE |
|------|-----------|-----------|-----------|
| 概览 | 第二节 概览 | 第二节 概览 | 第二节 概览 |
| 主营业务 | **第五节** "发行人主营业务情况" | **第二节** "发行人主营业务经营情况" | **第二节** "发行人主营业务经营情况" |
| 核心财务 | "主要财务数据和财务指标" | "发行人**报告期**的主要财务数据和财务指标" | "发行人**报告期内**的主要财务数据和财务指标" |
| 财务单位 | 元 | 万元 | 万元 / 亿元 |
| 风险因素 | 第三节 风险因素 | 第三节 风险因素 | 第三节 风险因素 |

**关键差异**：
- 北交所主营业务在 **第五节**（不是第二节），标题是"发行人主营业务情况"（无"经营"二字）
- 上交所/深交所主营业务在 **第二节 概览**，标题是"发行人主营业务经营情况"
- 三家财务数据章节标题略有差异 —— 用模糊搜索"主要财务数据和财务指标"定位

**抽取策略**（按章节名搜索，不要靠页码）:

1. **主营业务**：搜"发行人主营业务情况" 或 "发行人主营业务经营情况"，抽前 500 字
2. **财务数据**：搜"主要财务数据和财务指标"，找附近表格，**必抽**：营业收入、净利润、毛利率、净资产收益率
3. **风险因素**：搜"风险因素"，抽前 5 条风险标题
4. **⚠ 报告里绝不能写"未提取到"或"需进一步读取"**——概览找不到就翻全文找；全文还找不到就从叙述性文字里抽

**必须抽到的财务指标**（缺一项报告就是不完整的）:

| 指标 | 中文 | 格式 |
|------|------|------|
| 营业收入 | 营业收入 | X.XX 亿 或 XXXX 万 |
| 归母净利润 | 净利润 / 归属于母公司股东的净利润 | X.XX 亿 或 XXXX 万 |
| 毛利率 | 综合毛利率 | XX.XX% |

### 四、募集说明书（再融资）—— 概览章节

**结构与招股说明书类似**，但额外关注：

1. **本次发行方案**:
   - 发行方式（向特定对象发行 / 公开发行 / 配股 / 可转债）
   - 发行对象（再融资发行对象类型 + 数量限制）
   - 发行价格/定价基准日（如适用）
   - 锁定期安排
   - 发行规模上限（股数 + 金额）

2. **募投项目**:
   - 拟投资项目清单（项目名称 + 投资总额 + 拟用募集资金额 + 项目效益）
   - 募集资金必要性、合理性论证

3. **报告期财务数据**:
   - 最新一期 + 同业可比公司比较

4. **前次募集资金使用情况**（如有）

### 五、重组报告书（发行股份购买资产）—— 概览章节

**结构特殊，必抽要素**：

1. **本次交易方案**:
   - 交易方式（发行股份购买资产 + 募集配套资金 / 现金 + 股份 / 吸收合并 等）
   - 标的资产（公司名称 + 主营业务 + 主要财务数据）
   - 交易对方（交易对手身份 + 持股比例 + 关联关系）
   - 交易作价 + 评估值 + 评估方法（资产基础法 / 收益法 / 市场法）
   - 估值溢价率
   - 配套融资金额（如有）

2. **业绩承诺与补偿安排**:
   - 业绩承诺期（3 年或 4 年）
   - 各年承诺净利润额
   - 补偿方式（股份补偿 / 现金补偿 / 减值测试补偿）

3. **是否构成重组上市（借壳）**: 是 / 否

4. **过渡期损益安排 + 锁定期安排**

5. **同行可比交易案例**

## 注意事项

- **数据来源**：交易所公开页面，仅供个人学习/研究，**不做商业用途**
- **抓取频率**：不要高频（默认每两次请求间隔 0.5 秒）
- **上交所抓取**：首次需 `py -3.12 -m playwright install chromium`（已装则免）
- **回复要点拆解**：LLM 必须按 Q&A 配对结构输出，**禁止合并多个问题写成段落**
- **概览章节摘要**：必须按上面"必抽要素"逐项填写，**禁止写"未披露"或"需进一步查阅"——找不到就翻全文**
